## 이번에 논의하고 바꾼 것 (주제별, 결정 이유 중심)

### 1. 차트 UX
- 기간 버튼(3개월/전체)을 누르면 **데이터를 다시 거르지 않고 보이는 범위만 조정**하게 바꿈 → 축소하면 과거(2023~)가 더 나온다.
- 팝업을 **호버 → 클릭/터치 토글**로 변경 (마우스만 대도 뜨던 게 거슬려서).
- 캔들 가리던 **우측 축 라벨(MA·지지저항) 제거**, 크로스헤어의 무의미한 `00:00:00`도 제거.

### 2. 매매 시나리오 = 분할 매수 플랜  ★이번 세션 핵심 논의
- 원래 "진입 후보가는 지지선인데, 손절·목표·손익비는 현재가 기준"이라 **모순**이었음 → 전부 **진입가(평단) 기준**으로 통일.
- 단일 진입 → **검출된 지지선 전체(최대 4개)를 1·2·3·4차 분할 진입가**로.
- **비중(20%/30%…)은 일부러 안 넣음.** 이건 분석 도구지 실행 도구가 아니고, 자본·성향을 모르는데 비중을 정해주는 건 주제넘음. 대신 **균등 매수 평단가**만 참고로 표시.
- 1차 목표는 임의 +10%가 아니라 **실제 저항선**. 손익비가 낮게 나와도 그게 정직한 신호("저항이 코앞이라 단기 먹을 게 적다").

### 3. 가장 강한(확률 높은) 지지선 강조
- 강도 = **과거 테스트 횟수 + 60·120일선 겹침**. (가장 가까운 게 강한 게 아니라, 여러 번 테스트되고 버틴 게 강한 것.)
- **거래량은 뺐음** — 우리가 가진 건 일별 총거래량뿐이고, 제대로 쓰려면 가격대별 볼륨프로파일이 필요한데 없어서 노이즈만 늘어남.
- 가장 강한 지지선을 **금색 + ★핵심 지지**로 지지선 섹션·시나리오 카드·차트에 모두 표시.
- 처음엔 강한 지지선이 진입 단계에 안 들어가서 시나리오에 ★가 안 떴음 → **진입 단계에 포함**시켜 해결.

### 4. 데이터 갱신 + 신선도  (serve.py 새로 만듦)
- 대시보드는 **실시간이 아니라 빌드 시점 스냅샷**이라는 점을 명확히 하고,
  - **신선도 배지**("n분 전 갱신", 색상 구분)
  - **🔄 갱신 버튼 + 장중(평일 09:00~15:30) 5분 자동 갱신**
  을 추가. `serve.py`(로컬 서버, 포트 8765)로 띄워야 작동하고, 그냥 파일로 열면 버튼이 안내만 뜸(기존 방식 안 깨짐).

### 5. 자잘한 정합성 수정
- 매크로 설명 공식이 차원오류였음(× USD/INR → ÷ USD/INR로 정정).
- "52주 내 위치"가 전체 3년 기준으로 계산되던 것 → 실제 52주 기준으로.
- 추세분석에 **현재가·이평선 정렬 띠**(높은값→낮은값) 추가 — 정배열/역배열 한눈에.
