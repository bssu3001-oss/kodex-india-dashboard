## 1. 이게 뭐하는 프로젝트인가

KODEX 인도Nifty50 ETF(종목코드 **453810**)의 **기술적 분석 대시보드**.
네이버·야후에서 데이터를 받아 계산하고, 카카오스톡 같은 차트 + 시그널 + 매매 시나리오를
**하나의 정적 HTML 파일**로 만들어 브라우저에서 본다.

- **언어/의존성**: Python 3 표준 라이브러리만 사용 (pip 설치 불필요)
- **데이터 출처**:
  - 네이버 비공식 API — 일봉 캔들, 현재가/시세, NAV, 뉴스
  - 야후 파이낸스 — USD/KRW, USD/INR, Nifty50 지수
- **차트 라이브러리**: TradingView LightweightCharts v4.x (standalone, `vendor/`에 동봉)

⚠️ **실시간이 아님**: `build.py` 실행 시점의 스냅샷 데이터를 HTML에 구워 넣는 방식.
갱신하려면 다시 빌드해야 한다(아래 "갱신 기능" 참고).

## 2. 실행 방법 (두 가지)

### A. 그냥 한 번 보기 (기존 방식)
```bash
python3 build.py          # 또는 run_dashboard.command 더블클릭
```
→ 데이터 받아서 `dashboard.html` 생성하고 브라우저로 엶. 갱신 버튼은 안내만 뜸.

### B. 갱신 기능까지 쓰기 (서버 모드)
```bash
python3 serve.py          # 또는 대시보드_서버.command 더블클릭
```
→ `http://localhost:8765/dashboard.html` 로 열림.
→ **🔄 데이터 갱신** 버튼, **장중 자동 갱신(5분)** 작동.
→ 종료는 터미널에서 `Ctrl+C`.

## 3. 파일 구조와 역할

```
kodex-india-dashboard/
├── build.py              # 메인 빌드: fetch→indicators→signals→backtest→render→open
├── serve.py              # 로컬 서버 + POST /api/rebuild (port 8765)
├── run_dashboard.command # 더블클릭 실행기 (빌드 1회)
├── 대시보드_서버.command   # 더블클릭 실행기 (서버 모드)
├── templates/
│   └── dashboard.html    # ★ HTML+CSS+JS 템플릿 원본 (여기를 수정함)
├── lib/
│   ├── fetch.py          # 네이버·야후 데이터 수집 (stdlib urllib)
│   ├── indicators.py     # SMA·RSI·ATR·정배열·골든/데드크로스·지지저항·강도
│   ├── signals.py        # 종합 시그널 + 매매 시나리오(분할 매수 플랜)
│   ├── backtest.py       # 골든크로스 전략 백테스트
│   └── render.py         # 템플릿 + JSON 데이터를 dashboard.html로 합침
├── vendor/
│   └── lightweight-charts.standalone.production.js
├── data/last_good.json   # 네트워크 실패 시 폴백 캐시
└── dashboard.html        # ★ 생성물 (직접 수정 X, build.py가 덮어씀)
```

**중요**: 화면에 보이는 `dashboard.html`은 **생성물**이다.
실제 수정은 항상 **`templates/dashboard.html`** 에서 하고 `python3 build.py`로 다시 만든다.
데이터는 `render.py`가 `__DASHBOARD_DATA__` 자리에 JSON으로 박아 넣고,
브라우저 JS에서 전역 `D` 객체로 접근한다 (`D.indicators`, `D.signal`, `D.quote` 등).

## 5. 알아둬야 할 핵심 설계 결정 (왜 이렇게 했는가)

### 차트
- **데이터는 항상 전체를 로딩**하고, 기간 버튼(3개월/전체)은 `setVisibleRange`로 보이는 범위만 바꾼다. → 축소하면 과거가 더 보인다.
- `fixLeftEdge:true` + `fixRightEdge:true` → 데이터 양 끝에 딱 맞아 공백 없음.
- 거래량은 `priceScaleId:'vol'` + `scaleMargins{top:0.8}` 로 하단 20%에 깔았다 (`pane`은 v4에서 안 됨).
- 크로스헤어 `CrosshairMode.Magnet`, `timeVisible:false`(일봉이라 시각 불필요).
- 팝업은 `subscribeClick` 토글(다시 누르면 닫힘). 우측 축 라벨은 캔들 가려서 다 끔.

### 매매 시나리오 = 분할 매수 플랜 (signals.py)
- 검출된 **지지선 전체(최대 4개)를 1·2·3·4차 진입가**로 사용.
- **균등 매수 평단가** 기준으로 손절·목표·손익비 계산 (현재가 기준 X — 그건 모순이었음).
- **비중(allocation)은 일부러 안 넣음** — 분석 도구지 실행 도구가 아니라서. 평단가만 참고로 표시.
- 1차 목표 = 실제 **저항선** (임의 +10% 아님 → 정직한 손익비).
- 손절 = 가장 깊은 진입 ATR×2 아래 (그마저 깨지면 추세 이탈).

### 가장 강한 지지선 (indicators.py `support_resistance`)
- 강도 점수 = **과거 테스트 횟수**(±0.7% 밴드, 최근 120일) + **60·120일선 겹침**(±1.2%) 보너스.
  `score = touches + confluence*3`. 최고점이 `strongest_support`.
- **거래량은 일부러 제외** — 일별 총거래량뿐(가격대별 볼륨프로파일 없음)이라 노이즈만 늘어서.
- 지지선 섹션·시나리오 카드·차트 라인 **모두 금색 + ★핵심 지지**로 강조.
- (참고) 현재 데이터에선 가장 강한 게 가장 깊은 지지선(12,402, 20회 테스트)으로 나옴. 트레이딩상 "마지막 방어선"으로 해석 일관됨.

### 리스크 계산기
- 매수가 칸 기본값을 시나리오 평단(`sc.entry`=avg_entry)으로 프리필 → 두 카드 일관.
- 단, 계산기 1차목표는 일반 룰(+10%)이라 시나리오(저항선)와 손익비 숫자는 의도적으로 다를 수 있음.

### 갱신/신선도 (serve.py + templates JS)
- `serve.py` 가 정적 서빙 + `POST /api/rebuild`(build 재실행). 서버는 즉시 뜨고 초기 갱신은 **백그라운드**(야후 호출 지연 회피).
- 신선도 배지: `D.generated_at` 기준 "n분 전", 30초마다 갱신. 색상=신선(초록)/30분+(노랑)/장마감(회색).
- 자동 갱신: `localStorage`로 on/off 유지, **장중(평일 09:00~15:30)에만** 5분마다. `isMarketHours()`로 판정.
- **그레이스풀**: 파일/정적서버로 열면 `/api/rebuild` 없어서 갱신 버튼이 안내 알림만 띄움 → 기존 방식 안 깨짐.

## 8. 집에서 이어서 하려면

⚠️ **현재 git 원격(remote)이 없음.** 로컬 커밋만 있는 상태라 그대로는 다른 컴퓨터에서 못 받는다. 둘 중 하나:

**옵션 A — GitHub 등에 푸시 (권장)**
```bash
# 1) GitHub에서 빈 저장소 생성 후
git remote add origin <레포 URL>
git push -u origin main
# 2) 집 컴퓨터에서
git clone <레포 URL>
cd kodex-india-dashboard
python3 build.py   # 바로 실행됨 (pip 설치 불필요)
```

**옵션 B — 폴더 통째로 복사**
`kodex-india-dashboard/` 폴더를 USB·클라우드로 복사. `.git`까지 같이 옮기면 커밋 이력도 보존됨.
집에서 `python3 build.py`만 하면 동작 (의존성 없음).

집에서 환경 확인:
```bash
python3 --version   # 3.x 면 OK
python3 build.py    # 정상 빌드되면 준비 끝
```
